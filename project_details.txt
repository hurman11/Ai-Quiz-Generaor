===========================================================
AI QUIZ GENERATOR - PROJECT DETAILS
===========================================================

PROJECT OVERVIEW
----------------
This project is an "AI Quiz Generator" that allows teachers to upload educational material (PDF, Word, or text files), automatically generates a multiple-choice quiz using AI, and allows students to take the quiz in real-time. It also features an AI Tutor to help students understand their mistakes. 

The project is divided into a Python backend and a React/Next.js frontend.


===========================================================
PYTHON BACKEND (MAIN FOCUS)
===========================================================

1. LIBRARIES USED IN PYTHON (from backend/requirements.txt)
-----------------------------------------------------------
The backend is built primarily with Python. Here are all the libraries used and their purpose in the project:

- fastapi: The core web framework used to build the API. It is fast and modern.
- uvicorn[standard]: The server that runs the FastAPI application.
- groq: The AI library used to communicate with the Llama 3 model (Llama-3.3-70b-versatile) to generate the quizzes and power the AI Tutor.
- pydantic: Used for data validation. It ensures that the data sent to and from the API (like user registration details or quiz submission data) is in the correct format.
- python-dotenv: Used to securely load environment variables (like the Database URL and Groq API Key) from a .env file.
- python-multipart: Used to handle file uploads when the teacher uploads a PDF or Word document.
- PyPDF2: Used to read and extract text from uploaded PDF files.
- python-docx: Used to read and extract text from uploaded Word (.docx/.doc) files.
- psycopg2-binary: The PostgreSQL database adapter. It allows Python to connect to and interact with the Neon Postgres database.
- bcrypt: Used for security. It hashes user passwords before saving them in the database so they cannot be read by anyone.
- pyjwt: Used for authentication. It generates JSON Web Tokens (JWT) when a user logs in, keeping them securely logged in across requests.


2. WHERE AND HOW THE BACKEND CONNECTS WITH THE FRONTEND
-----------------------------------------------------------
The connection between the backend and frontend happens entirely over the network via an HTTP REST API. 

In the backend, this connection is defined in the file:
=> `backend/main.py`

This file contains all the "Endpoints" (URLs) that the frontend calls to send and receive data. The frontend connects to these endpoints using HTTP requests. Here are the specific connection points defined in `main.py`:

* Authentication (Connecting users to their accounts):
  - POST /auth/register: Frontend sends user details to create an account.
  - POST /auth/login: Frontend sends email/password, backend returns an access token.
  - GET /auth/me: Frontend requests the profile of the currently logged-in user.

* Quiz Generation & Management (Teacher Side):
  - POST /extract-text: Frontend uploads a file. Backend uses PyPDF2/python-docx to read it and returns the text.
  - POST /generate-quiz: Frontend sends the extracted text. Backend sends it to the Groq AI model and returns the generated quiz in JSON format.
  - POST /active-quiz: Frontend tells the backend to start a live quiz.
  - GET /results: Frontend frequently calls this to get real-time scores of students taking the quiz.

* Student Gameplay (Student Side):
  - GET /active-quiz: Frontend fetches the current live quiz data.
  - POST /submit-answer: Frontend sends a single answer as the student clicks it in real-time.
  - POST /submit-result: Frontend sends the final score when the student finishes.
  - POST /ai-tutor: Frontend sends the wrong answers, and the backend uses the Groq AI to return a personalized explanation for the student.


===========================================================
FRONTEND (MAIN THINGS ONLY)
===========================================================

As the frontend is optional, here is a brief overview of its core structure:

- Framework: Built using Next.js (a React framework).
- Language: Written in TypeScript (which is JavaScript with strict type-checking).
- Styling: Uses Tailwind CSS for rapid and modern styling.
- Connection to Backend: The frontend connects to the `main.py` backend by using the standard JavaScript `fetch()` function inside various React components (like `HostView.tsx`, `LiveStudentView.tsx`, and `QuizForm.tsx`). It sends HTTP requests to the backend URLs (e.g., `http://localhost:8000/generate-quiz`) and displays the data received.
